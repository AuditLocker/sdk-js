# Agent guide ŌĆö sdk-js

Operational guide for AI agents (Claude Code, Cursor, Aider, Codex CLI) and humans working in this repo. The README is for SDK _consumers_; this file is for SDK _contributors_. If you change the toolchain, update this file in the same commit.

## What this repo is

`@auditlocker/sdk` ŌĆö the JavaScript / TypeScript client, **generated by [Speakeasy](https://www.speakeasy.com/) from the AuditLocker OpenAPI spec**. `src/` is generated code. Do not hand-edit anything under `src/`, `docs/`, or the generated `*.md` (USAGE/FUNCTIONS/RUNTIMES/CONTRIBUTING) ŌĆö a regen overwrites it. Behaviour that can't be expressed in the spec lives in hooks and the facade; generation config lives in `.speakeasy/gen.yaml`.

## Repo layout

- `src/` ŌĆö **generated** SDK source (sdk/, funcs/, models/, hooks/, lib/, types/). Speakeasy-owned.
- `test/` ŌĆö hand-written vitest suite. `method-names.test.ts` is the method-name snapshot gate; `cjs-require.test.ts` proves the dual-emit CJS entry; `runtime-smoke.mjs` is the cross-runtime import smoke.
- `.speakeasy/gen.yaml` ŌĆö SDK-shaping config (generation-time decisions). Speakeasy rewrites/reorders it on every run; edit deliberate keys and re-run.
- `.speakeasy/workflow.yaml` ŌĆö source + target. Pins `speakeasyVersion`. Spec source and registry are maintainer configuration ŌĆö regeneration is a maintainer task (see `AGENTS.local.md` if you have it).
- `.speakeasy/gen.lock` ŌĆö generation checksums/feature versions. Committed.
- `package.json` ŌĆö Speakeasy owns name/version/type/exports/scripts(lint,build,prepublishOnly)/deps. Everything else is preserved via `additionalPackageJSON` / `additionalScripts` / `additionalDependencies` in `gen.yaml` ŌĆö **edit those, not package.json directly** (direct edits are wiped on regen).
- `pnpm-workspace.yaml` ŌĆö all pnpm settings (pnpm 11 reads `.npmrc` for auth/registry only), supply-chain gates, `packages: ['.']` (required, see gotcha), and transitive-security `overrides`. Not Speakeasy-managed; edit directly.
- `pnpm-lock.yaml` ŌĆö committed lockfile with integrity hashes and tarball URLs.
- `tsconfig.json` ŌĆö Speakeasy-generated; `tshy` drives the build.
- `.oxlintrc.json` ŌĆö Speakeasy-generated, tuned for generated code. `.oxfmtrc.json` ŌĆö hand-written; ignore-lists all Speakeasy-managed paths (we don't reformat generated code).
- `audit-ci.jsonc` ŌĆö audit-ci config. Allowlist is empty by design (see Supply-chain).
- `.nvmrc` ŌĆö Node 24 (dev/CI). `LICENSE` ŌĆö Apache 2.0.

## Toolchain

Toolchain: oxlint / oxfmt / tsgo (no ESLint / Prettier / tsc).

| Concern            | Tool                                               | Why                                                                        |
| ------------------ | -------------------------------------------------- | -------------------------------------------------------------------------- |
| Package manager    | **pnpm 11** (pinned via `packageManager`)          | `minimumReleaseAge` + `strictDepBuilds` + `trustPolicy` supply-chain gates |
| SDK generator      | **Speakeasy CLI** (pinned in `workflow.yaml`)      | Spec-driven; near-daily releases, so the version is pinned, never floating |
| Build              | **tshy**                                           | Dual CJS+ESM emit from one TS source (Speakeasy-managed)                   |
| Lint               | **oxlint**                                         | Speakeasy default for generated code; upstream toolchain parity            |
| Format             | **oxfmt**                                          | Upstream parity; only formats hand-written files (generated is ignored)    |
| Type checker       | **tsgo `--noEmit`** (`@typescript/native-preview`) | Upstream parity; runs clean on the generated SDK                           |
| Tests              | **vitest 4**                                       | Fast, first-class TS                                                       |
| Vuln scanner       | **audit-ci v7** via `pnpm dlx`                     | Runs BEFORE install in CI                                                  |
| Package validation | **publint** + **attw**                             | Exports + type-export correctness before publish                           |

## Daily commands

```bash
nvm use                                   # Node 24 (.nvmrc)
pnpm install

# Gates (CI runs all of them; Build before Test ŌĆö the CJS smoke needs dist/)
pnpm lint                                 # oxlint
pnpm format                               # oxfmt --check  (format:fix to write)
pnpm typecheck                            # tsgo --noEmit
pnpm build                                # tshy ŌåÆ dist/esm + dist/commonjs
pnpm test                                 # vitest
pnpm publint                              # package.json validation
pnpm attw                                 # type export correctness (node16 profile)
pnpm run audit:ci                         # audit-ci (moderate+ threshold)
```

## Regenerating the SDK

SDK regeneration (`speakeasy run`) is a **maintainer task** ŌĆö it needs the
Speakeasy workspace API key and the upstream OpenAPI spec. Maintainers: see
`AGENTS.local.md`. Contributor-relevant invariants:

- **`speakeasy run --skip-versioning`** keeps `version` at `0.0.0`
  (pre-release, npm-name reservation only) ŌĆö automatic versioning would
  otherwise churn it.
- **Always re-install after a regen** (`pnpm install`), then run the gates ŌĆö
  a regen rewrites `package.json` (see the gotcha below).
- **Method names / SDK shape are controlled in the spec, never here.** Clean
  method names (`events.ingest()`, not `events.ingestEvent()`) come from
  `x-speakeasy-name-override` on the API routes. If a regen changes the
  public surface, `test/method-names.test.ts` fails ŌĆö fix via the spec
  override, never by editing the snapshot or renaming an `operationId`
  (public forever).
- **Bumping the Speakeasy pin** (`workflow.yaml` `speakeasyVersion`) is its
  own reviewed change: re-verify the documented placement claims against the
  new version's docs, record what changed, then bump.

## Supply-chain gates ŌĆö DO NOT weaken without discussion

Settings live in `pnpm-workspace.yaml` with inline comments ŌĆö read it directly. Procedures not in the config:

- **Bypassing `minimumReleaseAge` for a critical CVE:** add the package to `minimumReleaseAgeExclude`. Log the exemption in the PR.
- **audit-ci runs before `pnpm install` in CI** ŌĆö `pnpm dlx`, not a devDependency.
- **Dependabot cooldown** (`.github/dependabot.yml`): 7-day cooldown on all tiers.
- **npm provenance:** `publishConfig.provenance: true` (preserved via `additionalPackageJSON`). Requires `id-token: write` in the publish job.
- **`audit-ci.jsonc` allowlist is empty by design.** Transitive advisories are remediated by forcing patched versions in `pnpm-workspace.yaml` `overrides` (currently `vite ^8`, `postcss >=8.5.10` ŌĆö the SDK never runs a Vite dev server, but we patch rather than allowlist). Adding an allowlist entry is per-advisory risk acceptance and is NOT done without discussion.

## Gotchas

### Dual CJS + ESM (not ESM-only)

`moduleFormat: dual` in `gen.yaml`. tshy emits `dist/esm/` + `dist/commonjs/`, each with an inner `package.json` (`{"type":"module"}` / `{"type":"commonjs"}`); top-level stays `"type":"module"`. `attw` uses the `node16` profile (the SDK requires Node Ōēź22, so node10 resolution failures are correctly ignored ŌĆö do **not** revert to `--profile esm-only`). The dual-emit contract is regression-tested by `test/cjs-require.test.ts`, which asserts the resolved CJS path genuinely routes through `/commonjs/` (Speakeasy uses plain `.js` in both dirs, so a `.cjs` suffix check would not fire).

### `packages: ['.']` is required in `pnpm-workspace.yaml`

Single-package repo, but the key is **not** optional: `verifyDepsBeforeRun: error` validates against pnpm's workspace-state file, which pnpm only writes when a workspace is declared. Without `packages: ['.']` every `pnpm run`/`exec` fails `ERR_PNPM_VERIFY_DEPS_BEFORE_RUN` unconditionally. Do not remove the key.

### `overrides` live in `pnpm-workspace.yaml`

pnpm 11 stopped reading the package.json `pnpm` field (it warns and ignores `pnpm.overrides`), so transitive-security overrides live in `pnpm-workspace.yaml` `overrides`. The old pnpm#9777 interaction with `verifyDepsBeforeRun: error` that previously forced them into package.json is fixed in pnpm 11. They survive regen because Speakeasy never rewrites `pnpm-workspace.yaml`. Do not move them back to package.json ŌĆö pnpm 11 will silently drop them.

### Re-install after every `speakeasy run`

A regen rewrites `package.json` (deps via `additionalDependencies`, scripts via `additionalScripts`). `verifyDepsBeforeRun: error` then refuses every script until `pnpm install` re-validates. Workflow is always: `speakeasy run` ŌåÆ `pnpm install` ŌåÆ gates.

### `strictDepBuilds` will bite on new deps with postinstall

`"ignored build scripts: <pkg>"` ŌåÆ review the script, add to `allowBuilds` in `pnpm-workspace.yaml` (`<pkg>: true` to run it, `false` to acknowledge-and-skip). **Never** set `strictDepBuilds: false`.

### Single approved runtime dep: Zod v4 mini

`dependencies` carries **only** `zod` (Speakeasy validates requests/responses with it at runtime; `zodVersion: v4-mini` in `gen.yaml`). Every other runtime dep ships in the customer's app and expands their attack surface ŌĆö adding one is a deliberate PR-level decision with the dep audited. Treat every Zod major bump as a deliberate PR (regenerate + test the published bundle); minor/patch flow through Dependabot's cooldown.

### `engines.node` is `>=22`

Node 20 EOL'd 2026-04-30; the SDK has zero published versions, so the floor was set to `>=22` before 1.0 (risk-free now; a breaking change after). `.nvmrc` stays on 24 (dev/CI parity). The CI matrix is `[22, 24]`. `engines` is preserved via `additionalPackageJSON`. Next re-evaluation trigger: Node 22 EOL (~2027-04). Do not use Node-Ōēź24-only APIs in hand-written `test/` without a guard.

### `pnpm audit` vs `pnpm run audit:ci`

`pnpm audit` ŌĆö quick local feedback, no allowlist/threshold. `pnpm run audit:ci` ŌĆö the exact CI invocation (threshold + config from `audit-ci.jsonc`). Script is `audit:ci` (not `audit`) to avoid shadowing the built-in.

## When this guide goes stale

If you upgrade pnpm across a major version, re-verify: `minimumReleaseAge` integer minutes; `strictDepBuilds` default-refuses; `blockExoticSubdeps` transitive-only; `trustPolicy: no-downgrade` exists; `verifyDepsBeforeRun: error` + the `packages: ['.']` interaction; `overrides` honoured; `pnpm audit --json` shape matches audit-ci v7; `pnpm publish --provenance` emits Sigstore. If any change, update this file and `pnpm-workspace.yaml` together.

If you bump the Speakeasy pin, re-verify the four documented placement claims (pagination/retries/errors positions, `gen.yaml` typescript defaults) against the new version's docs before committing.
